## 1. 产品概述
为用户提供一个极简的个人浏览器起始页，聚合常用网站快捷入口，并以加密方式安全存储各网站登录凭证。数据本地加密存储，通过 GitHub Gist 实现跨设备同步。

### 1.1 产品定位
- 个人工具型产品，非商业级密码管理器
- 纯前端 SPA，零后端依赖
- 强调极简、快速、隐私安全

### 1.2 目标用户画像
- 日常需要访问多个网站的白领 / 开发者
- 对隐私敏感，不希望密码存在云端明文
- 有基本的 GitHub 使用能力

## 2. 功能需求

### 2.1 主密码保护 (P0)
| ID | 功能 | 说明 |
|----|------|------|
| AUTH-01 | 首次设置主密码 | 首次使用时引导用户设置主密码，密码强度要求 ≥8 位，含字母+数字 |
| AUTH-02 | 解锁验证 | 每次打开页面/刷新后，需输入主密码解锁才能查看内容 |
| AUTH-03 | 密码派生加密 | 主密码通过 PBKDF2 派生 AES-256-GCM 密钥，用于加密所有敏感数据 |
| AUTH-04 | 退出锁定 | 关闭页面标签即锁定，再次打开需重新输入主密码 |

### 2.2 网站快捷入口管理 (P0)
| ID | 功能 | 说明 |
|----|------|------|
| LINK-01 | 添加入口 | 添加网站：名称、URL、所属分组。图标自动通过 Google Favicon 获取 |
| LINK-02 | 编辑入口 | 修改已保存网站的名称、URL、分组 |
| LINK-03 | 删除入口 | 删除网站入口及其保存的账号密码 |
| LINK-04 | 快捷跳转 | 点击网站卡片直接在新标签页打开 |
| LINK-05 | 搜索过滤 | 按名称/URL 关键词实时过滤网站列表 |
| LINK-06 | 拖拽排序 | 支持在同组内拖拽调整顺序 |

### 2.3 分组管理 (P0)
| ID | 功能 | 说明 |
|----|------|------|
| GRP-01 | 创建分组 | 自定义分组名称，如"工作"、"社交"、"开发" |
| GRP-02 | 编辑/删除分组 | 删除分组时提示迁移组内网站 |
| GRP-03 | 分组折叠 | 点击分组标题可折叠/展开该组 |

### 2.4 密码管理 (P0)
| ID | 功能 | 说明 |
|----|------|------|
| PWD-01 | 保存账号密码 | 为每个网站记录用户名/密码，加密后存储 |
| PWD-02 | 查看密码 | 主密码解锁状态下可查看明文（需点击"显示"） |
| PWD-03 | 一键复制 | 点击复制用户名/密码到剪贴板，复制后有短暂 toast 提示 |
| PWD-04 | 编辑凭证 | 可修改已保存的账号密码 |
| PWD-05 | 清空凭证 | 删除该网站保存的凭证数据 |

### 2.5 数据同步 (P1)
| ID | 功能 | 说明 |
|----|------|------|
| SYNC-01 | 配置 GitHub Token | 用户配置 GitHub Personal Access Token |
| SYNC-02 | 配置 Gist ID | 指定用于同步的 Gist ID（支持新建和关联已有） |
| SYNC-03 | 手动推拉同步 | 提供"推送"和"拉取"按钮，用户手动触发同步 |
| SYNC-04 | 冲突处理 | 同步时以最新操作为准，推送前提示可能覆盖远端数据 |

### 2.6 数据管理 (P1)
| ID | 功能 | 说明 |
|----|------|------|
| DATA-01 | 导出加密备份 | 导出加密后的完整数据为 JSON 文件 |
| DATA-02 | 导入恢复 | 从加密备份文件导入恢复数据 |
| DATA-03 | 重置所有数据 | 清除所有本地数据重新开始 |

## 3. 交互流程

### 3.1 首次使用流程
```
打开页面 → 设置主密码（含强度校验）→ 进入空状态主页 →
引导添加第一个网站 → 填写名称/URL/分组 → 添加成功
```

### 3.2 日常使用流程
```
打开页面 → 输入主密码解锁 → 查看网站列表 →
  ├─ 点击卡片 → 新标签页跳转
  ├─ 点击卡片上的钥匙图标 → 查看/复制账号密码
  ├─ 搜索框输入 → 实时过滤列表
  └─ 点击添加 → 新增入口
```

### 3.3 同步流程
```
用户点击"推送" → 加密当前数据 → PUT → GitHub Gist API
用户点击"拉取" → GET → GitHub Gist API → 解密 → 比对时间戳 → 覆盖本地
```

## 4. 数据模型

```json
{
  "version": 1,
  "updatedAt": "2026-05-19T12:00:00Z",
  "salt": "base64_salt",
  "groups": [
    { "id": "g1", "name": "工作", "order": 0 },
    { "id": "g2", "name": "社交", "order": 1 }
  ],
  "sites": [
    {
      "id": "s1",
      "groupId": "g1",
      "title": "GitHub",
      "url": "https://github.com",
      "order": 0,
      "credentials": {
        "username": { "encrypted": "base64...", "iv": "base64..." },
        "password": { "encrypted": "base64...", "iv": "base64..." }
      }
    }
  ]
}
```

### 4.1 说明
- **salt**: 主密码派生密钥时使用的随机盐值
- **credentials**: 每个凭证字段独立加密，各自有独立 iv
- 每个网站可仅保存 URL 不保存密码，credentials 可为空

## 5. 技术方案

### 5.1 技术栈
| 层 | 技术选型 | 说明 |
|----|---------|------|
| 框架 | Vue 3 (Composition API) | 组件化开发 |
| 构建 | Vite 6 | 快速 HMR，小型构建 |
| 状态管理 | Pinia | 轻量状态管理 |
| 路由 | Vue Router | 用于解锁页/主页等视图切换 |
| 加密 | Web Crypto API | 浏览器原生加密能力 |
| 同步 | GitHub REST API | Gist 增删改查 |
| 图标 | Google Favicon API | 自动获取网站图标 |
| 部署 | GitHub Pages / Vercel | 静态部署 |

### 5.2 架构概要
```
┌─────────────────────────────────────┐
│            Vue 3 App                │
├─────────────────────────────────────┤
│  Pages:  LockPage / MainPage        │
├─────────────────────────────────────┤
│  Components: SiteCard / GroupPanel  │
│              SearchBar / SyncPanel   │
├─────────────────────────────────────┤
│  Composables:                       │
│   ├─ useAuth     (主密码验证)       │
│   ├─ useCrypto   (加密/解密)        │
│   ├─ useStorage  (localStorage CRUD)│
│   └─ useSync     (GitHub Gist)      │
├─────────────────────────────────────┤
│  Store (Pinia): appStore            │
├─────────────────────────────────────┤
│  localStorage: 加密后的 JSON 数据    │
└─────────────────────────────────────┘
```

### 5.3 加密方案详解

```
主密码
   │
   ▼
PBKDF2 (600,000 次迭代, SHA-256)
   │
   ▼
AES-256-GCM Key ──→ 加密每个 credentials 字段
   │                  (每个字段独立随机 iv)
   ▼
localStorage: { salt, encryptedData }
```

- **PBKDF2 迭代次数**: 600,000 次（OWASP 2023 推荐）
- **加密模式**: AES-256-GCM（提供认证加密，防篡改）
- **主密码不存储**: 派生 salt 存储，主密码用完即销毁
- **每次关闭标签页自动锁定**: sessionStorage 不持久化密钥

### 5.4 GitHub Gist 同步方案

```
推送:
  本地加密数据 → PUT /gists/{id} → Gist 更新

拉取:
  GET /gists/{id} → 解析文件内容 → 本地覆盖

认证:
  Authorization: Bearer ghp_xxxxx (用户配置的 PAT)
```

- 同步触发方式：手动（点击按钮）
- Token 存储：加密后存入 localStorage
- 冲突策略：以用户主动操作为准（推送覆盖远端，拉取覆盖本地）

### 5.5 部署方案
- 构建输出静态文件（dist/）
- 可部署至 GitHub Pages、Vercel、Netlify、Cloudflare Pages
- 不需要服务器，纯静态托管

## 6. 非功能需求

| 维度 | 要求 |
|------|------|
| 性能 | 首屏加载 < 1s（无框架运行时仅 ~100KB） |
| 安全 | 所有敏感数据 AES-256-GCM 加密存储 |
| 兼容 | 支持 Chrome / Edge / Firefox 最新版本 |
| 可用 | 核心操作（打开、搜索、跳转）3 步内完成 |
| 隐私 | 零数据上传到第三方（除 GitHub Gist 同步） |
